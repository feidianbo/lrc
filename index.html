介绍

项目中包括了 3 个文件

  • .css 要自定义 配合 js 就可以了
  • main.js
  • index.html 演示文件

灵动的力量

将js 文件导入播放器页面，注意默认的是HTML5 中的audio 或 vedio 标签，如果是其他，还需要使用之配置一个适配器了 一般的将文件最后导入 为妙，
引入之后，只需要 4 行代码就可搞定

  • lcz.set;rc_objec('歌词显示标签的id')
  • lcz.mf_Init();
  • lcz.load(lcz.Read('歌词原格式'，解析器)，回调函数，也就是解析完必之后的第一个处理函数)
    • 与播放器挂钩： 在播放器的 ontimeupdate 事件中加入一个函数 lcz.torun();

其次：

工具中自动 带了 两个解析器：

1. lcz.Read(); 默认解析器： 解析一般歌词 就是单时间格式的 [00:12.30]歌词 2. lcz.vls1(); 加强型： 更高级点 可以解析多时间歌词 [00:00.45][10.50.20].. 歌词

同样的有两个 时间追踪器

1. lcz.getLrcByVlue(var); // 相对于两者，反应较慢 （默认） 2. lcz.getLrcByValueInd(var); //相对于两者，反应快 （在歌曲快进，快退是表现明显）

当让还有 其他 可以设置 哦

          lcz.classV1="v1"; //一号className(原始) 
       lcz.classV2="v2"; //二号className (设定)
     	lcz.classV3="v3"; //三号className  (另类)
     	lcz.initTop = 200;  //初始化显示位置 200 px
     	lcz.center = 150;   //歌词焦点中心位置   50 px 
     	lcz.empty = 20;   //初始化构建多少个 空对象 （ 同时表示 头和尾）
     	lcz.isDropLrc = true;  //是否启用本地歌词 拖入功能
     		// 解读歌词过程中用到的 标记
     	lcz.tag = {
         	sname :"ti",  //歌曲名
         	cname :"cl",  //作词
         	qname :"cs",  //作曲
         	bname :"ps",   //编曲
         	sgname :"ar",    //歌手名称
         	special :"al",   //所属专辑
         	kname :"by",		//歌词制作
         	end :"end"    //结束 语
     	};
